Can you think of an analogy to describe HTML,CSS and the DOM? How would you describe it to a non-technical friend?

An analogy would be a script for a show or play.
HTML elements are the lines for the characters. e.g. person A:"a line of dialogue"
CSS provides stage directions. e.g. preson A speaks loudly while crossing the sceen.
DOM is the format for how to adress parts of the show. e.g. Act 3 Scene 2 Line 5

What is meant by boxifying design?

Seperating a design out into areas/boxes of distinct things. For a web page a few examples of boxes are heading, navigation and content. Within these boxes they can often be further seperated into smaller areas such as blog entires and then further still with boxes like text and images.

What is the box model?

When talking about a website from a visual design perspective the box model refers to how CSS shows and treats an element. each box has the box itself which is represented with the border attribute. for outside the box there is the attribute margin which tells the box how to deal with other boxes around it. Inside the box is the content which is the substance of the box which is then told how it fits within the box with the padding attribute which says how to stay away from the boarder of the box.
